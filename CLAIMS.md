# MIRAGE — Claims for Artifact Evaluation

This document enumerates the quantitative claims made in the MIRAGE paper
(*VehicleSec '26*) and the expected tolerance for each when the artifact is
re-executed. An automated checker is provided at
[`scripts/verify_results.py`](scripts/verify_results.py).

> **Source.** All reference values are taken from the paper as submitted,
> aggregated over five seeds with randomized traffic conditions (σ = 0.5,
> varied initial speeds and gaps). See paper §7 for the experimental protocol.

## Evaluation setup assumed by all claims

| Parameter | Value |
|---|---|
| Simulator | OMNeT++ 5.6.2 + Veins 5.2 + SUMO 1.8.0 + VASP |
| Scenario | 3-lane highway, 20 vehicles, 120 s sim time (paper §6.1) |
| Seeds | 5 (`highway_seed{0..4}.rou.xml`, generated by `generate_route_variants.py`) |
| Attacker | `node[19]` (rearmost vehicle) |
| Operating threshold Θ | 0.55 |
| Detection granularity | per-BSM |
| Total BSM evaluations per seed | ≈ 4.7 M (3.7 M benign + 1.0 M attack) |

Claims below report mean ± standard deviation across the 5 seeds unless
otherwise stated.

## C1 — Aggregate Effectiveness (paper §7.1, Table 4)

**MIRAGE achieves F1 = 0.922 ± 0.001 at Θ = 0.55**, with TPR = 0.883 ± 0.003,
FPR = 0.009 ± 0.001, and Precision = 0.965 ± 0.007.

| Metric | Reference | Tolerance |
|---|---|---|
| F1 | 0.922 | ±0.015 |
| TPR | 0.883 | ±0.020 |
| FPR | 0.009 | ±0.005 |
| Precision | 0.965 | ±0.015 |

**Verified by.** `verify_results.py --claim C1` reads
`results/detlog-P_{Baseline,A1,A2}-*-*.csv` for the five seeds, computes the
aggregate four-way metrics, and compares them against the table above.

**Tolerance rationale.** Seed variance (five seeds) combined with OMNeT++'s
scheduler tie-breaking on identical timestamps can shift per-BSM classification
counts by ~0.5 %. A ±0.015 band on F1 comfortably covers this while still
catching any substantive regression.

**Auto-scaling for fewer seeds.** `verify_results.py` automatically widens the
tolerance bands by $\sqrt{5/N}$ when the reviewer runs with $N < 5$ seeds
(e.g.\ Tier 2 `run_single_seed.sh`). This reflects the standard statistical
fact that the variance of a mean scales as $1/N$. At $N=1$, bands are
widened by $\sqrt{5} \approx 2.24\times$; at $N=5$ the scaling is $1.0$.
Pass `--no-scale` to force paper-exact tolerances regardless of $N$.

## C2 — Balanced Per-Attack Coverage (paper §7.3, Figure 5)

**MIRAGE is the only detector that sustains high TPR on both attack variants.**

| Attack | MIRAGE TPR (reference) | Tolerance |
|---|---|---|
| A1 (NoStop, `attackType=10`) | 0.770 ± 0.006 | ±0.030 |
| A2 (WithStop, `attackType=11`) | 0.997 ± 0.000 | ±0.010 |

Reviewer-observable contrast: no baseline detector exceeds TPR = 0.25 on *both*
attacks. VCADS (B2) = 0.997 on A1 but 0.000 on A2; Threshold (B1) = 0.218 on
A1 but 0.976 on A2.

**Verified by.** `verify_results.py --claim C2` filters rows from
`detlog-P_A1-*.csv` and `detlog-P_A2-*.csv` by `attack_type` and computes TPR
per variant. The reviewer output should show both values in the tolerance
bands above, and the per-attack bar pattern in `reference_results/figures/
fig3_per_attack.png` should be reproduced qualitatively.

## C3 — Baseline Comparison (paper §7.2, Table 4)

**MIRAGE substantially outperforms all three baselines in aggregate F1.**

| Detector | Reference F1 | Tolerance | `detectorType` |
|---|---|---|---|
| B1 Threshold | 0.733 ± 0.004 | ±0.020 | 1 |
| B2 VCADS | 0.607 ± 0.018 | ±0.040 | 2 |
| B3 F2MD / VASP | 0.193 ± 0.006 | ±0.020 | 3 |
| **MIRAGE (Proposed)** | **0.922 ± 0.001** | **±0.015** | **4** |

**Verified by.** `verify_results.py --claim C3` reads detlogs for all four
detectors (`B1_*`, `B2_*`, `B3_*`, `P_*`) and reports the ranking. The critical
check is the ordering `F1(P) > F1(B1) > F1(B2) > F1(B3)`; each absolute value
is additionally checked against its tolerance band.

## C4 — Threshold Robustness (paper §7.4, Figure 6)

**MIRAGE is robust to the exact choice of Θ within a stable band:**

> *Performance is stable across Θ ∈ [0.45, 0.59]: TPR remains above 0.88 and
> F1 above 0.92.* (paper §7.4)

| Θ | Expected F1 (lower bound) | Expected TPR (lower bound) |
|---|---|---|
| 0.45 | ≥ 0.92 | ≥ 0.88 |
| 0.52 | ≥ 0.92 (best F1, nominal 0.925) | ≥ 0.88 |
| 0.55 | ≥ 0.92 (chosen operating point) | ≥ 0.88 |
| 0.59 | ≥ 0.92 | ≥ 0.88 |

Outside this band, the paper reports a sharp transition: at Θ = 0.60, TPR
drops to 0.41 and F1 to 0.57 (Check 4 no longer exceeds threshold); at Θ =
0.45, FPR rises toward 0.05.

**Verified by.** Re-running at multiple Θ requires re-compilation because
Θ is a compile-time `constexpr` in `MisbehaviorDetectors.h`. Reviewers who
want to reproduce the sensitivity sweep can follow
`docs/threshold_sweep.md` (see that file for the recipe); the default
configuration uses Θ = 0.55 and reviewers who only rerun the default should
confirm C1 (F1 ≥ 0.907 after tolerance) and verify qualitatively that
`reference_results/figures/fig5_sensitivity.png` matches the expected stable
region.

## C5 — IDM Mitigation (paper §7.5)

**The IDM mitigation layer reduces mean braking severity by ≈ 75 % compared
to the undefended baseline.**

| Condition | Reference mean |a| (m/s²) |
|---|---|
| No defense (baseline) | see `reference_results/tables/tab_mitigation.tex` |
| MIRAGE with IDM mitigation | bounded at −4.0 m/s² |

**Verified by.** `verify_results.py --claim C5` reads the `mitigated_a`
column from `detlog-P_A{1,2}-*.csv` and the raw `a` column from the
corresponding `B0_*` runs, then compares mean absolute values. Target
reduction is 75 % ± 10 %.

Qualitative comparison: the reviewer plot should broadly match
`reference_results/figures/fig_mitigation.png`.

## What is *not* claimed

- Real-time performance on embedded hardware. All simulation runs are
  wall-clock faster than real time on commodity x86_64, but no claim is made
  about execution inside a production OBU.
- Detection under packet loss or lossy channels. The evaluation assumes
  ideal channel conditions; §8 (Limitations) discusses this.
- Detection under attacks other than A1 / A2 fake-EEBL. Other misbehavior
  classes (position-offset, dimension-offset, etc.) are not part of these
  claims.

## How to verify

Minimum re-run (single seed, ~10 min on a modern workstation):

```bash
./scripts/install.sh all                            # one-shot install, ~12 min
./scripts/run_single_seed.sh 0                      # all 18 configs, seed 0
./scripts/verify_results.py --all --seeds 0         # compare vs claims above
```

Full re-run (5 seeds, ~1 hour):

```bash
./scripts/run_multi_seed.sh 0 1 2 3 4
./scripts/verify_results.py --all --seeds 0 1 2 3 4
```

The verifier exits 0 when every enabled claim is within tolerance, and
non-zero otherwise, so it can be dropped into CI.
